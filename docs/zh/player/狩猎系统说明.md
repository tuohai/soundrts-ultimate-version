# 狩猎系统说明

SoundRTS 提供帝国时代风格的狩猎：村民攻击野生动物，击杀后在原地留下可采集的食物尸体，并可驱赶羊群跟随。

---

## 1. 玩家流程

1. **右键动物** → 村民自动攻击（强制命令，可打中立动物）
2. **击杀后** → 原地生成 `food_carcass`（食物尸体矿床）
3. **自动采集** → 若工人开启 `auto_gather`，会采集尸体并运回仓库
4. **受击逃跑** → 鹿、羊被攻击后会逃离；野猪会反击
5. **牧羊**（可选）→ 配置了 `can_herd 1` 的工人可驱赶 `herdable` 动物（如羊）

---

## 2. 语音播报与「动物」标识

狩猎动物虽由 `computer_only ... neutral` 放置，但**不会**播报为「中立 NPC」。

| 场景 | 播报示例 |
| --- | --- |
| 选中鹿 | 鹿 , 动物 |
| 方格摘要 | , 2鹿 , 动物 |
| Ctrl+Shift+F4 切到纯动物玩家 | 你是动物 |

识别规则：

- 单位配置了 `is_huntable 1` 或 `herdable 1` → 视为野生动物，播报「动物」
- 名称与「动物」之间用逗号隔开（与敌人、盟友等播报格式一致）
- Ctrl+Shift+F4 仅当该玩家**所有存活单位**均为野生动物时播报「你是动物」；混合 `quest_npc` 与鹿的玩家仍播报「你是中立 NPC」

剧情 NPC（`quest_npc` 等）保持「中立 , NPC」播报，与狩猎动物区分。

## 3. 地图放置

在地图文件中使用中立电脑玩家放置：

```txt
computer_only 0 0 neutral b3 4 deer 2 sheep
```

随机地图会在出生点附近自动生成浆果丛（`orchard`）与野生动物。

### 3.1 外交：动物不是联盟成员

狩猎动物由 `computer_only` 生成，但**不会**加入默认的 `ai` 电脑联盟，也**不能**与玩家或其它势力结盟。

| 规则 | 说明 |
| --- | --- |
| 识别条件 | 该 `computer_only` 槽位里**只有**配置了 `is_huntable 1` 或 `herdable 1` 的单位（如鹿、羊、自定义老虎） |
| 引擎行为 | 该槽位对应电脑的 `alliance = None`，`allied` 仅含自己 |
| 多群动物 | 每一行 `computer_only` 是独立狩猎点，**彼此之间也不结盟** |
| 混编槽位 | 同一行若既有动物又有步兵等，整槽仍按普通电脑处理，会进 `ai` 联盟 |
| 玩家外交 | 中立玩家不可 F12 结盟；狩猎动物更不会被当作可结盟势力 |

自定义动物示例（会按动物隔离，不进 `ai` 联盟）：

```txt
def tiger
class soldier
is_huntable 1
...

computer_only 0 0 neutral 5,5 2 tiger
```

若要做「多群动物联合成一个自然阵营」，需用触发器 `(alliance …)` 显式设置，不属于默认狩猎行为。

## 4. rules.txt 配置

### 4.1 内置单位

| 类型 | 说明 |
| --- | --- |
| `deer`（鹿） | 35 食物，受击逃跑 |
| `sheep`（羊） | 25 食物，可驱赶，受击逃跑 |
| `boar`（野猪） | 50 食物，会反击 |
| `food_carcass` | 狩猎后留下的食物矿床（`collision 0`，不占格） |
| `orchard` | 浆果丛/果园矿床，采集后运回基地计入食物 |

村民 `can_gather` 已包含 `food_carcass`（及 `orchard`）。

### 4.1.1 储量播报名与入库资源

狩猎尸体在地图上仍按来源动物播报物体名（如「鹿的尸体」）；果园等矿床用 `style.txt` 里的 `title`（如「果园」）。

「**含有 50 ___**」中数量后面的词由 `style.txt` 的 **`qty_unit_title`** 配置（尸体矿床默认「肉」、果园默认「果子」），与入库后玩家资源栏读法分开：

- `鹿的尸体 , 向东 , 含有 50 肉`
- `果园 , 向北 , 含有 50 果子`

运回含 `resource3` 的仓库后，资源栏与 **Shift+Z** 仍播报**食物**（`resource3_title`），由 rules 的 `resource_type resource3` 决定。

```txt
def food_carcass
class deposit
resource_type resource3

def orchard
class deposit
resource_type resource3
```

在 `ui/style.txt` 中可为矿床类型设置 `qty_unit_title`（对应 `ui-zh/tts.txt` 文案），仅影响「含有 N …」的播报，不改变尸体/果园物体名本身。

### 4.2 动物属性

| 属性 | 说明 |
| --- | --- |
| `is_huntable 1` | 可被村民狩猎；右键默认攻击 |
| `flee_on_hit 1` | 受击后向远离攻击者方向逃跑 |
| `herdable 1` | 可被 `can_herd` 工人驱赶跟随 |
| `herd_leash_range` | 驱赶跟随的最大距离（毫米） |
| `food_deposit` | 死亡后生成的食物矿床类型名 |
| `food_deposit_qty` | 尸体食物储量（游戏内单位，如 35 = 35 食物） |
| `no_number 1` | 同类型仅 1 个时不报序号（适合动物） |

工人侧：

| 属性 | 说明 |
| --- | --- |
| `can_herd 1` | 可驱赶 `herdable` 动物；默认 `0` |

### 4.3 自定义动物示例

```txt
def wolf
class soldier
is_huntable 1
flee_on_hit 1
food_deposit food_carcass
food_deposit_qty 40
no_number 1
ai_mode guard
...
```

### 4.4 狩猎科技

`hunting_techniques`（狩猎技术）：缩短尸体/浆果采集时间，提高尸体采集量，并增加动物尸体储量。可在市政厅研究。

## 5. 与中立 NPC 的区别

| | 狩猎动物 | 剧情 NPC |
| --- | --- | --- |
| 典型单位 | `deer`、`sheep`、`boar` | `quest_npc`、`npc_knight` |
| 识别属性 | `is_huntable` / `herdable` | 无（可有 `receive_items`） |
| 语音归属 | 动物 | 中立 , NPC |
| 玩家主动攻击 | 需强制攻击（与中立 creep 相同） | 需强制攻击 |

详见 [单位默认模式与自动状态配置说明.md](单位默认模式与自动状态配置说明.md)。
