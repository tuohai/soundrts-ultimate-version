# 随机地图功能说明

SoundRTS 1.4.3.4 起提供**程序化随机地图**（Random Map Generator，简称 RMG）：按菜单选项生成标准 `.txt` 地图，与手工地图走同一加载流程，可用于本地练习或联机建房。

---

## 1. 入口

| 场景 | 路径 |
| --- | --- |
| 本地练习 | 主菜单 → **开始游戏** → **随机地图**（列表第一项） |
| 联机建房 | 连接服务器 → 创建对局 → 地图列表中选择 **随机地图** → 选速度 → 配置参数 |

配置完成后，本地模式会进入「邀请电脑 / 选阵营 / 开始」流程；联机模式向服务器发送 `create_random` 命令，由主机在开局时生成地图。

---

## 2. 配置流程

菜单按顺序引导以下选项（可随时 **Esc** 返回上一级）：

1. **地图模板**（或 **导入分享码**，见第 4 节）
2. **地图尺寸**：小 / 中 / 大
3. **玩家数量**：2 / 3 / 4
4. **团队模式**（仅 4 人）：自由混战（FFA）或 2v2 固定组队
5. **野怪强度**：弱 / 中 / 强（地图中央敌对守军，会主动攻击玩家；弱 2 步兵 / 中 4 步兵+2 弓箭手 / 强 6 步兵+4 弓箭手+1 骑士）
6. **资源布局**：均衡 / 集中
7. **地形**（通道模板除外）：随机 / 草地 / 沼泽 / 山地
8. **水域**（通道模板除外）：无 / 湖泊 / 河流
9. **宝箱**：无 / 少 / 多（需当前 rules 中定义 `class item` 的可拾取物品）
10. **胜利模式**：征服 / 经济 / 探索 / 生存
11. **种子**：随机，或输入自定义数字（0–99999）
12. **停火条约**：0 / 5 / 10 / 15 / 20 分钟

选完种子后会**语音预览**当前配置摘要；确认条约后生成地图。

### 2.1 地图模板

| 模板 | 说明 |
| --- | --- |
| **标准** | 经典方格地图，随机出生点与桥梁 |
| **快速** | 更高起始资源，适合快节奏对局 |
| **宏观** | 更高人口上限与更多草地，偏运营 |
| **通道** | 三行通道式布局（类似 TD2），无地形/水域选项，资源仍可选 |

### 2.2 团队 2v2

4 人且选 **2v2** 时，地图会写入结盟触发器：玩家 1 与 2、玩家 3 与 4 开局为盟友。

### 2.3 胜利模式

| 模式 | 胜利条件 | 开局目标语音（摘要） | 说明 |
| --- | --- | --- | --- |
| **征服** | 歼灭所有**敌方玩家** | 歼灭所有敌方玩家 | 默认模式；**不必**清地图中央野怪或兵营守卫 |
| **经济** | **累计采集**达到目标金币（不含开局自带） | 累计采集达到 N 黄金 | 花掉已采金币仍算进度；配置预览会播报 N；约每 60 秒检测一次 |
| **探索** | 亲访每一处古代遗迹 | 亲访每一处古代遗迹 | 须**亲自**进入每一处遗迹（敌方先探不算你的进度）；2v2 队友探过的计入；首次全图奖励仍只发一次；开局 **seed 任务简报**；发现时播报剩余数量 |
| **生存** | 倒计时结束时仍保有主基地 | 坚守 N 分钟并保留主基地 | 快速 10 分钟，其余 15 分钟；丢主基地先失败；多人可同时获胜 |

**经济模式目标金币**（仅 `resource1`）：

| 地图模板 | 目标 |
| --- | --- |
| 快速 | 2000 |
| 标准 | 3000 |
| 宏观 | 5000 |
| 通道 | 2500 |

所有模式均会因失去所有 `provides_survival` 建筑（主基地等）而失败。探索、经济、生存模式下**不会**因「全灭敌人」自动胜利，但仍可主动进攻。

所有模式均会在地图中生成**古代遗迹**（首次进入获得资源）和**可占领兵营**（先清守卫，再攻击兵营一次即可占领；占领后可训练步兵/弓箭手；未占领时会定期增援守卫）。

**探索模式额外体验**（与其它模式的差异）：

- **任务简报**：进入对局后（邀请电脑并开始后），会先播放由 **种子决定** 的简短任务简报（任务简报 + 变体提示 + 探索胜利目标），同一分享码每次相同。
- **遗迹两步发现**：所有胜利模式下，进入遗迹后会提示检查**相邻方格**；若该遗迹旁有可用邻格，再派单位进入邻格可获得**额外资源**（深处另有额外收获）。
- **探索进度**：仅在探索模式下，每次发现遗迹后会语音播报尚未发现的遗迹数量；只剩一处时会单独提示。

## 3. 生成结果与 F5/F6

本地生成完成后会立即播报：

- **地图已生成**
- **种子**（数字，可用于复现同一地图）
- **分享码**（完整配置字符串）
- **按 F5 重复朗读**（历史提示）

随后进入邀请电脑菜单时，上述播报**不会丢失**：按 **F5** 重复上一条消息，按 **F6** 逐条浏览语音历史，可随时回听种子与分享码。

菜单内同样可用 **F5 / F6** 浏览历史（与游戏内一致）。

## 4. 分享码

### 4.1 格式

分享码形如：

```text
RMG1:f:m:2:med:b:r:f:v:hi:4242
```

冒号分隔 **12 段**：`RMG1` 前缀 + **11** 个参数字段（旧版 10 段分享码无「胜利模式」字段，导入后默认为征服）：

| 字段 | 含义 | 取值示例 |
| --- | --- | --- |
| 模板 | standard / fast / macro / lanes | `s` / `f` / `m` / `l` |
| 尺寸 | small / medium / large | `s` / `m` / `l` |
| 玩家数 | 2–4 | `2` |
| 野怪 | weak / medium / strong | `w` / `med` / `s` |
| 资源 | balanced / clustered | `b` / `c` |
| 地形 | random / grass / marsh / mountain | `r` / `g` / `a` / `t` |
| 团队 | ffa / teams_2v2 | `f` / `t` |
| 水域 | none / lake / river | `n` / `l` / `v` |
| 宝箱 | none / low / high | `n` / `lo` / `hi` |
| 胜利模式 | conquest / economic / exploration / survival | `c` / `e` / `x` / `s` |
| 种子 | 0 表示随机；>0 为固定种子 | `4242` |

粘贴时可带或不带 `RMG1:` 前缀；也接受 `/` 代替 `:` 作为分隔符。

### 4.2 导入分享码

在 **地图模板** 子菜单第一项选 **导入分享码**，在输入框中粘贴或键入分享码，**Enter** 确认，**Esc** 取消。

输入框支持标准编辑快捷键（与种子、登录名等 `input_string` 输入框相同）：

| 快捷键 | 作用 |
| --- | --- |
| **Ctrl+A** | 全选 |
| **Ctrl+C** | 复制（无选区时复制全部） |
| **Ctrl+X** | 剪切 |
| **Ctrl+V** | 粘贴（自动过滤非法字符） |
| **Backspace / Delete** | 删除选区或光标前/后字符 |

分享码最长 80 字符；只允许字母、数字及 `:` `/` `.` `-`。

导入成功后会播报配置预览并直接进入 **停火条约** 菜单（跳过中间逐步选项）。无效分享码会提示 **分享码无效** 并返回模板菜单。

---

## 5. 联机说明

- 房主在随机地图菜单完成配置后，服务器收到 `create_random …` 命令；地图在**开局时**由主机按相同种子与参数生成，保证各客户端一致。
- 联机流程中**不会**在客户端重复播报本地那种「地图已生成 + 分享码」；如需分享配置，请在建房前记录分享码，或让对方导入同一分享码建房。
- 公开房、条约分钟数等仍按普通建房流程在速度/公开性子菜单中选择。

---

## 6. 与旧版「随机」的区别

地图文件里的 `#random_choice` / `#end_random_choice` 仍是**预处理随机**，在固定地图骨架上随机放置资源等，不是 RMG。

RMG 是**整图程序化生成**：尺寸、出生点、资源、野怪、水域、宝箱等均由参数决定，并可用种子与分享码复现。
