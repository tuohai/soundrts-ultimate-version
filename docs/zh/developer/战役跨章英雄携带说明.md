# 战役跨章英雄携带说明

单人战役支持把**指定英雄**的进度带到下一关。全部由 **`rules.txt`** / **`campaign.txt`** 配置，**不硬编码**单位名（如雷诺）。合作战役不支持（避免联机不同步）。

玩家向简述见 [../player/战役与合作战役改进说明.md](../player/战役与合作战役改进说明.md) §2.1.1。  
剧情物品与 `campaign_flag` 见 [给NPC物品功能说明.md](给NPC物品功能说明.md)、[../player/战役密信与结盟说明.md](../player/战役密信与结盟说明.md)。

---

## 1. 三种跨章机制对照

| 机制 | 配置位置 | 携带内容 | 典型用途 |
| --- | --- | --- | --- |
| **`campaign_carryover`** | `rules.txt` 单位字段 | 等级+经验、背包（可拆分） | 主角 RPG 式成长 |
| **`campaign_flag`** | 地图触发器 | 布尔剧情状态 | 结盟、支线完成 |
| **`add_inventory_item`** | 地图触发器 | 指定物品进背包 | 信物、钥匙、下一关补发 |

三者可单独使用或组合。例如：只开 `campaign_carryover_stats`，信物仍用 `campaign_flag` + 开局 `add_inventory_item`。

---

## 2. `rules.txt` 字段

写在**最初声明**跨章的主角 def 上（子版本 `is_a` 继承即可）。

| 字段 | 默认 | 说明 |
| --- | --- | --- |
| `campaign_carryover` | `0` | `1` = 启用跨章存档（总开关） |
| `campaign_carryover_id` | def 名 | `campaigns.ini` 键名前缀 `hero_<id>_` |
| `campaign_carryover_stats` | `1`（在 carryover 开启时） | `1` = 保存/恢复 **等级+经验** |
| `campaign_carryover_inventory` | `1`（在 carryover 开启时） | `1` = 保存/恢复 **背包** |

### 2.1 示例：完整继承（默认）

```text
def my_hero
is_a knight
campaign_carryover 1
inventory_capacity 8
max_level 20
xp_threshold_growth linear 100 50
hp_max_per_level 20
```

（仍可使用手写 `xp_thresholds 200 500 1000`。）

通关时写入 `hero_my_hero_xp` / `_level` / `_inventory`；下一关恢复三者。

### 2.1a 开局等级与经验（`level` / `xp`）

与 [`doc_src/src/zh/modding.rst`](../../../doc_src/src/zh/modding.rst) **Heroes** 一节相同，可在英雄 def 上写：

| 字段 | 说明 |
| --- | --- |
| `level` | 开局等级（默认 `1`）。`> 1` 时生成单位时逐级应用 `*_per_level` 与 `level_skills` |
| `xp` | 开局累计经验（可选） |
| `level 0` | 从 0 级开始；状态播报「等级 0，经验 0/首个 xp_thresholds」 |

跨章恢复仍优先于地图默认：存档等级与 `hero_min_level` 取较大值后，再按 `_per_level` 补回累计成长。

### 2.2 只带经验/等级，不带背包

```text
def my_hero
is_a knight
campaign_carryover 1
campaign_carryover_inventory 0
inventory_capacity 8
```

下一关背包以地图 `starting_units` 为准，等级与经验仍继承。

### 2.3 只带背包，不带经验/等级

```text
def my_hero
is_a knight
campaign_carryover 1
campaign_carryover_stats 0
inventory_capacity 8
```

下一关保留地图默认等级，只恢复背包物品。

### 2.4 完全不跨章携带

不写 `campaign_carryover 1`（或写 `0`）。每关英雄状态完全由地图决定。

### 2.5 多章节单位版本共用存档

```text
def my_hero
campaign_carryover 1
campaign_carryover_id my_hero

def my_hero_ch3
is_a my_hero_ch2
; 无需重复写 campaign_carryover
```

地图第 3 关放 `my_hero_ch3`，存档键仍为 `hero_my_hero_*`。

## 3. `campaign.txt`：最低等级

可选一行，进入某章及之后时强制英雄**至少**该等级（与存档等级取较大值）：

```text
hero_min_level 13:2 16:3 19:4 22:5 25:6 27:7
```

格式：`章节号:等级`，空格分隔。雷诺战役示例见 [`res/single/The Legend of Raynor/campaign.txt`](../../../res/single/The Legend of Raynor/campaign.txt)。

---

## 4. 存档格式（`user/campaigns.ini`）

```ini
[The_Legend_of_Raynor]
chapter = 5
difficulty = standard
flags = ch24_garrek,ch24_garrek_token
hero_raynor_xp = 1200
hero_raynor_level = 3
hero_raynor_inventory = sword,health_potion
```

| 键 | 含义 |
| --- | --- |
| `hero_<id>_xp` | 经验（仅 `campaign_carryover_stats 1` 时写入） |
| `hero_<id>_level` | 等级 |
| `hero_<id>_inventory` | 逗号分隔的物品 `type_name` 列表 |
| `flags` | `campaign_flag` 集合（与英雄存档独立） |

- **仅胜利**时更新；失败重打不覆盖。
- 多个 `campaign_carryover_id` 不同的英雄可同时存在（少见）。

---

## 5. 运行时流程

```
地图加载 → populate_map
    → restore_hero_on_world（读 campaigns.ini，按 rules 恢复 stats/inventory）
    → 对局进行
    → 胜利 → save_hero_after_victory（按 rules 写入 stats/inventory）
```

源码：[`soundrts/campaign_hero.py`](../../../soundrts/campaign_hero.py)  
调用：[`soundrts/game.py`](../../../soundrts/game.py)（加载后 restore、胜利后 save）

## 6. 与触发器配合

### 6.1 自动背包 vs 剧情补发

| 场景 | 推荐 |
| --- | --- |
| 装备、药水等「整包继承」 | `campaign_carryover_inventory 1` |
| 必须保证下一关有某剧情物 | `set_campaign_flag` + 下一关 `(timer 0) (add_inventory_item …)` |
| 只记录「拿过信物」不继承其它 | `campaign_carryover_inventory 0` + flag |

北方密信（24→25 章）同时使用：**雷诺 `campaign_carryover 1`** + **`ch24_garrek_token`** 开局补发 `garrek_token`。

### 6.2 相关触发器

| 触发器 | 作用 |
| --- | --- |
| `(set_campaign_flag 名)` | 跨章永久标记 |
| `(campaign_flag 名)` | 检查标记 |
| `(add_inventory_item 物品 [数量] [单位type])` | 放入背包；省略单位时找有背包的单位 |
| `(unset_campaign_flag 名)` | 清除标记（重玩单章） |

`map_flag` / `set_map_flag` **不跨章**。

---

## 7. 限制

| 项目 | 说明 |
| --- | --- |
| 合作战役 | 不 restore / 不 save；`campaign_flag` 亦为 no-op |
| 经验与等级 | `campaign_carryover_stats` 同时控制二者 |
| 消耗型物品 | `consume_on_pickup 1` 的物品不进背包，无法被继承 |
| 多个 carryover 英雄 | 支持，每种 `campaign_carryover_id` 各存一套 |
