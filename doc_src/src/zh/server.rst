独立服务器指南
==============

.. contents::

简介
----

本指南说明如何安装独立公共服务器。公共服务器允许任何玩家创建游戏。


从源码安装
----------

- 压缩包方式：
    - 从 https://github.com/soundmud/soundrts/releases/latest 下载最新版本的源码压缩包
    - 在你选择的文件夹中解压。
- Git 方式：
    - 安装 v1.3.2：
        - git clone https://github.com/soundmud/soundrts.git
        - git checkout v1.3.2
    - 之后更新到 v1.3.3：
        - git fetch
        - git checkout v1.3.3
- 在主文件夹中创建一个名为 "user" 的空文件夹。
- 安装 Python 3.7 或更高版本。
- 启动服务器：python server.py
- 这将生成 "user/SoundRTS.ini"。
- 按 Control+C 关闭服务器。
- 编辑新创建的 "user/SoundRTS.ini"：
    - 将 "login" 设为服务器名称。
    - 若希望只允许至少 2 名人类玩家的游戏，将 "require_humans" 设为 1。
- 启动服务器：python server.py
- 确保你的服务器可从外部访问。


如何让服务器可从外部访问
------------------------

在大多数情况下，你需要在路由器中配置端口转发，将 2500 端口的入站 TCP 连接转发到服务器的本地 IP 地址。

你可能还需要在路由器中配置 DHCP，以确保服务器始终拥有相同的本地 IP 地址。

如果你位于防火墙之后，可能需要确保允许 2500 端口的入站 TCP 连接。


如何检查服务器是否可从外部访问
------------------------------

要检查服务器是否可从本地网络外部访问，可等待一名玩家连接，或最好请朋友从外部连接。

作为最后手段，你也可以使用端口转发测试网站（例如搜索 "port forwarding tester"）。请谨慎：我无法保证这类网站不是恶意的！网站不应要求你安装工具，等等。


服务器列表
----------

服务器列表由元服务器（metaserver）托管。

服务器启动后，应会自动加入列表。这并不意味着你的服务器可从外部访问。

服务器停止后，会自动从列表中消失，但可能需要一些时间。


SoundRTS.ini 中的 require_humans 参数
-------------------------------------

默认值：0

若 require_humans 设为 1，则至少有两名人类玩家注册之前，服务器不允许游戏创建者邀请电脑。

只有公共服务器受此参数影响。
