# 星际 mod：人族附属建筑与重组说明

适用模组：``mods = starcraft`` （在 ``SoundRTS.ini`` 中启用）。

本文说明 Tech Lab / Reactor 等附属建筑（addon）的建造、起飞分离、飞行重组，以及「草地」与「插槽对齐」的区别。


----


1. 基本概念
-----------



.. list-table::
   :header-rows: 1

   * - 概念
     - 说明
   * - 宿主建筑
     - 可挂附件的主建筑：兵营、工厂、星港（``can_have_addon``）
   * - 附属建筑
     - Tech Lab、Reactor（``is_addon 1``），建在宿主侧面插槽
   * - 插槽
     - 宿主坐标 ``(x, y)`` 东侧约 3.5 格（``addon_offset_x``，默认 3500 内部单位）
   * - 草地
     - 同格内的 ``meadow`` 对象；地面人族建筑降落时必须消耗一块草地
   * - 重组
     - 宿主起飞后附件留在地面；另一宿主飞过来、插槽对齐后降落，自动挂接孤立附件



Tech Lab 按宿主解锁训练/研究，例如：

- 兵营 + Tech Lab → 劫掠者
- 工厂 + Tech Lab → 攻城坦克
- 星港 + Tech Lab → 医疗运输机

Reactor 提供 ``addon_train_multiplier 2`` （双倍训练队列）。


----


2. 建造附属建筑
---------------


1. 选中已有宿主（如兵营），不要单独点空地。
2. 在建造菜单选择 Tech Lab 或 Reactor。
3. 附件在宿主侧面自行建造（``self_constructs 1``），不占单独草地。

测试图：``terran_addon_test`` （联机菜单，启用 starcraft mod）。


----


3. 起飞（Lift-off）
-------------------


兵营 / 工厂 / 星港可变形为飞行形态（``can_change_to flying_*``）：

1. 选中地面建筑 → 菜单 变形 → 对应飞行单位。
2. 主建筑离地；附件留在原格，与宿主解除关联（``detach``）。
3. 原位置生成一块新草地（``Meadow``），供日后降落占用。

同一格可因此出现多块草地（例如兵营、工厂各起飞一次 → 两块草地）。出生点若只有 1 块地图草地，被兵营占用后，工厂可能落在无草地位置；起飞后才会在各自原位置出现草地。


----


4. 普通降落 vs 重组降落
-----------------------


4.1 两套独立判定
~~~~~~~~~~~~~~~~



.. list-table::
   :header-rows: 1

   * - 阶段
     - 决定什么
     - 不决定什么
   * - 降落
     - 是否消耗草地；工厂落在哪个 (x, y)
     - 是否对接附件
   * - 对接
     - 孤立 Tech Lab 是否挂接到工厂
     - 用的是哪块草地



草地 = 落地许可（同格必须有 meadow 可删）。  
插槽 = 宿主 ``(x,y)`` 与附件 `` (x+3500, y)`` 的几何关系；对接只看插槽是否在容差内（约 2.5 格 曼哈顿距离）。

因此可能出现：工厂视觉上占的是「中心草地」，但逻辑坐标已在插槽上，仍能对接并训练坦克——因为插槽对齐，不是因为「站在实验室脚下那块草上」。

4.2 普通降落（落回自己的起飞草地）
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


1. Tab 选中该格目标草地（起飞后生成的那块）。
2. 退格（默认 ``go``）飞过去，等到达。
3. 菜单 变形 回地面形态。

结果：工厂落在该草地对应位置，不会接管同格孤立的 Tech Lab。若同格仍有可对接的实验室，会语音提示：需要靠近科技实验室再降落才能对接（TTS 7350）。

4.3 重组降落（接管孤立 Tech Lab）
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


1. 先在兵营上建造 Tech Lab，再起飞兵营（实验室留在地面）。
2. 起飞工厂，飞到该格。
3. Tab 科技实验室（不是 Tab 草地）。
4. 退格 go — 系统会把目标改到宿主降落插槽（实验室西侧约 3.5 格），而不是实验室本体中心。
5. 到达后 变形 为工厂。

结果：

- 工厂生成在插槽坐标；
- 消耗的是离插槽最近的同格草地（常常是中心的某块草地）；
- 插槽与 Tech Lab 对齐 → 自动挂接 → 可训练攻城坦克等。

测试图：``terran_recombine_test``；战役 ``sc_build_tests`` 第四章。

5. 操作速查
-----------



.. list-table::
   :header-rows: 1

   * - 目的
     - Tab 目标
     - go 之后
     - 变形结果
   * - 工厂落回起飞点
     - 工厂起飞留下的草地
     - 退格
     - 不对接，不能训 Tank
   * - 工厂接管 Tech Lab
     - 科技实验室
     - 退格（飞向插槽）
     - 对接成功，可训 Tank



同格多块草地时，Tab 循环听到的都是「草地」，请靠方向区分；重组务必 Tab 实验室。

6. 常见现象
-----------


Q：实验室旁边没有草地，只有中心两块草地，go 到实验室降落后工厂在中心草地上，为什么还能训 Tank？

A：go 到 Tech Lab 时实际飞向插槽；降落时工厂 (x,y) 在插槽，只是删除的是离插槽最近的那块中心草地。对接检查的是 ``Tech Lab`` 与 ``工厂.x + 3500`` 的距离，与「用的是哪块草地」无关。

Q：为什么以前落在中心草地也能对接？

A：旧版在同格内距离实验室过远时仍会强制吸附到兵营侧插槽。现已改为：go 到自己草地则优先落原地；重组必须 go 到 Tech Lab。

Q：插槽对齐但距离看起来远？

A：实验室在宿主侧面约 3.5 格，不在宿主正下方；这是星际争霸式布局，属正常。

7. 地图与 mod 作者（rules.txt 关键字）
--------------------------------------



.. list-table::
   :header-rows: 1

   * - 关键字
     - 用途
   * - ``can_have_addon``
     - 宿主可挂的附件类型列表

   * - ``addon_host_types``
     - 附件可挂在哪些宿主上

     - 挂接后额外训练单位
   * - ``addon_grants_research``
     - 挂接后额外研究
   * - ``addon_train_multiplier``
     - Reactor 倍数
   * - ``can_change_to`` / ``ground_form``
     - 起飞与降落形态
   * - ``change_time``
     - 变形时间
