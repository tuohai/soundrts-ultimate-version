音频管理系统（P0–P2 重构）
==============================

1.4.5.1 对客户端音频引擎做了分阶段重构。**P0 / P1 / P2 指重构阶段**（结构性整理 → 体验优化 → 听感打磨），**不是**喊杀音效的三层分类，也**不是** ``psounds.play(..., priority=…)`` 里的通道抢占优先级数值。

喊杀分层（``shout_bg`` / ``shout_unit`` / ``shout_event``）见 :doc:`battle-shouts`。

.. contents::

架构概览
--------

::

  clientmain → clientmedia.init_media()
    ├─ sound.init()           pygame.mixer，channel 0 保留给语音
    ├─ sounds.load_default()  索引 ui/ 音效，按需解码
    └─ voice.init()           语音队列 + TTS

  运行时：
    channel 0      → UI 语音 / 旁白（VoiceChannel）
    channel 1..N   → 游戏 SFX（SoundManager / psounds）
    music stream   → 背景音乐（pygame.mixer.music）

核心模块：

+---------------------------+------------------------------------------+
| 模块                      | 作用                                     |
+===========================+==========================================+
| ``lib/sound.py``          | 混音、立体声定位、通道分配、BGM 状态     |
| ``lib/sound_cache.py``    | 资源分层索引、懒加载、预加载             |
| ``lib/music_resolver.py`` | 菜单/对局/战斗/胜负音乐与短音效查找      |
| ``lib/battle_music.py``   | 进战/停战音乐状态机                      |
| ``lib/voice.py``          | 语音播报队列                             |
| ``clientmedia.py``        | 音量调节入口                             |

P0：结构性重构
--------------

**目标**：减少 ``sound.py`` 单体膨胀、切换 Mod 时内存可控、消除类级可变状态隐患。

**音乐解析模块化（``music_resolver.py``）**

- 将 ``_find_music_file()`` 及地图/战役包查找逻辑集中到独立模块。
- 共享 helper：``find_map_or_campaign_asset()``、``get_active_context()``、``clear_music_cache()``。
- ``play_game_music``、``play_battle_music``、``play_victory_sound``、``play_defeat_sound`` 等入口共用同一查找链，删除数百行重复代码。

**SFX 缓存显式回收（``sound_cache.clear_decoded()``）**

- ``load_default()`` 切换 mod / 战役 / 地图时：先 ``psounds.stop()``，清空限流时间戳，再重建资源层。
- 避免旧解码样本长期挂在播放源上造成内存尖峰。

**类级可变状态修复**

- ``SoundSource``、``LoopingSoundSource``、``SoundManager`` 的 ``channel``、``_sources``、``_start_time`` 等改为实例属性，避免单例下状态泄漏。

**测试**：``test_music_resolver.py``

P1：体验与性能
--------------

**独立游戏音效音量（``sfx_volume``）**

- 配置项 ``audio/sfx_volume``（默认 0.5），写入 ``SoundRTS.ini``。
- **语音 / UI** 仍用 ``main_volume``；**战场音效**（脚步、战斗、环境循环等）用 ``sfx_volume``。
- 对局命令 ``cmd_sfx_volume``；菜单与对局均可调。
- 旁观追帧静音时同时静音 ``main_volume`` 与 ``sfx_volume``。

**语音等待事件化**

- ``voice._say_now()`` / ``flush()`` 不再 ``time.sleep(0.1)`` 阻塞主循环，改为 ``pygame.event.pump()`` + ``Clock.tick(30)`` 轮询。
- API 行为不变（仍等到播完），但 BGM 与输入响应更顺畅。

**菜单音乐 fallback 统一**

- ``_play_menu_music_with_fallback()`` 合并战役选关、建局、联机大厅三处重复 fallback 逻辑。

**测试**：``test_audio_settings.py``、``test_voice_pump.py``

P2：听感打磨
------------

P2a — 环境音平滑
~~~~~~~~~~~~~~~~

- ``ambient_volume()`` 不再每帧 ``random.random()``，改为基于坐标的慢速 LFO（约 8 秒周期）。
- 同一位置左右声道平滑变化；不同坐标相位不同，多路环境音不会完全同步闪烁。

P2b — 战斗音乐状态机（``battle_music.py``）
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- 集中处理：视野内是否有非中立敌人、进入/退出战斗音乐、玩家阵营设置。
- 原先分散在 ``combat.py``、``game_display.py``、``game_navigation.py`` 的重复侦察逻辑统一为：

  - ``enter_battle()`` — 发现新敌人
  - ``maybe_stop_battle()`` — 定期巡检停战
  - ``set_battle_mode()`` — 战斗事件触发
  - ``on_place_cleared()`` — 当前区域清空后再查全局

P2c — ``music_resolver`` 清理
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- 删除不可达死代码分支，补充音乐查找单测。

P2d — SFX 多格式与热音效预加载
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**多格式支持**

``ui/`` 下的游戏音效现支持：

- ``.ogg``（优先）
- ``.wav``
- ``.mp3``

同一 stem 多种格式并存时优先 ``.ogg``。引用时可写 ``footstep``、``footstep.wav`` 或 ``1029.mp3``，引擎会自动去后缀匹配。

**热音效预加载**

- 资源加载后，后台线程读取常用音效字节（默认含 ``1029`` beep、``1003`` 位置提示音，以及 ``parameters.d["preload_sounds"]`` 列表）。
- 主线程每帧 ``tick_preload()`` 解码（默认每帧 4 个），符合 pygame 线程安全要求。
- 对局在 ``game_display`` 循环解码；主菜单在 ``clientmenu`` 循环解码。

可在 ``parameters.txt`` 自定义::

  preload_sounds 4303,4304
  preload_decode_per_tick 6

**测试**：``test_ambient_stereo_volume.py``、``test_battle_music.py``、``test_sfx_formats.py``

Mixer 缓冲与采样率
------------------

启动时从 ``SoundRTS.ini`` 读取并传给 ``pygame.mixer.pre_init``（路径一般为
``user/SoundRTS.ini``，或 Windows 上的 ``%APPDATA%/SoundRTS/SoundRTS.ini``）::

  [audio]
  mixer_buffer = 2048
  mixer_frequency = 44100

  [general]
  num_channels = 16

+-------------------+---------+--------------------------------------------------+
| 配置键            | 默认值  | 说明                                             |
+===================+=========+==================================================+
| ``mixer_buffer``  | 2048    | SDL 缓冲样本数。越大对局越稳、延迟略增。合法值： |
|                   |         | ``512`` / ``1024`` / ``2048`` / ``4096`` /       |
|                   |         | ``8192``（其它值会靠到最近的一档）。             |
+-------------------+---------+--------------------------------------------------+
| ``mixer_frequency`` | 44100 | 输出采样率 Hz（``22050``–``48000``）。           |
+-------------------+---------+--------------------------------------------------+
| ``num_channels``  | 16      | 同时播放的音效通道数（对局很挤时可试 ``32``）。  |
+-------------------+---------+--------------------------------------------------+

44100 Hz 下常见缓冲大致延迟：``1024``≈23ms（对局忙时易卡）、``2048``≈46ms（默认）、
``4096``≈93ms（仍卡再试）。改完须**重启游戏**才生效（mixer 不能中途重初始化）。
旧 ini 没有这两项时，下次启动会自动补上默认值。

模组作者须知
------------

**音效文件**

- 放在 ``ui/`` 或 ``mods/<mod>/ui/`` 下。
- 推荐仍提供 ``.ogg`` 以减小体积；``.wav`` / ``.mp3`` 可直接使用，无需手动转换。
- 若 ``ui/tts.txt`` 中已有同 ID 文本条目，对应音效 ID 不会被加载为声音。

**音乐与短音效**

- 菜单/对局/战斗/胜负音乐查找支持 ``.mp3``、``.ogg``、``.wav``（见 ``music_resolver.MUSIC_FORMATS``）。
- 地图或战役包可在 ``ui/music/``、``ui/sounds/`` 下放置覆盖文件。

**与喊杀分层的关系**

+----------------------+--------------------------------+---------------------------+
| 概念                 | 含义                           | 文档                      |
+======================+================================+===========================+
| P0–P2 重构阶段       | 引擎模块化、音量、格式、预加载 | 本文档                    |
+----------------------+--------------------------------+---------------------------+
| 喊杀三层             | ``shout_bg`` / ``shout_unit`` / ``shout_event`` | :doc:`battle-shouts` |
+----------------------+--------------------------------+---------------------------+
| 通道 priority 数值   | ``psounds.play`` 抢占低优先级源 | ``lib/sound.py`` 实现注释 |

三者互不替代；发行说明中曾将 P0–P2 误写为「环境层 / 战斗层 / 提示层」优先级，已在 1.4.5.1 版本说明中更正。
